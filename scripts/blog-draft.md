> 이 글도 Claude Code가 작성했습니다.

## 바이브코딩이 뭔데?

요즘 개발자들 사이에서 "바이브코딩(Vibe Coding)"이라는 말이 돌고 있다. 한마디로 **AI한테 말로 시키면서 개발하는 것**이다. 내가 직접 코드를 작성하는 게 아니라, AI와 대화하면서 원하는 걸 만들어내는 방식이다.

나는 개발자다. 코드를 보면 손이 근질거린다. 근데 이번엔 **끝까지 참았다.** 직접 코딩하고 싶은 마음을 억누르고, 오직 말로만 시켜보는 실험을 해봤다. 결론부터 말하면 — **진짜로 된다.**

## 가장 큰 문제 — AI는 대화를 기억 못 한다

바이브코딩을 해보면 바로 벽에 부딪히는 게 있다. Claude는 세션이 끊기면 **대화 내용이 사라진다.** 코드 파일은 읽을 수 있으니 현재 상태는 파악할 수 있다. 하지만 "왜 이 방식을 선택했는지", "어떤 걸 시도했다가 실패했는지", "지금 어느 단계까지 왔는지" — 이런 **결정 과정과 진행 맥락**은 코드만 봐서는 알 수 없다.

### 힌트는 Shrimp Task Manager에서 얻었다

**Shrimp Task Manager**는 Claude에 연결할 수 있는 MCP(Model Context Protocol) 도구다.

> MCP란? Claude가 외부 도구(파일, DB, API 등)를 직접 연결해서 쓸 수 있게 해주는 표준 프로토콜이다.

Shrimp는 거대한 작업을 단계별 태스크로 쪼개고, 각 태스크의 진행 상태를 **외부 데이터 파일에 저장**한다. 덕분에 세션이 끊겨도 다음날 다시 시작할 때 저장된 파일을 읽어서 "어느 태스크까지 완료됐는지" 바로 파악하고 이어서 작업할 수 있다.

여기서 핵심을 하나 얻었다. **컨텍스트를 파일로 남겨두면 세션이 끊겨도 이어갈 수 있다.** MCP 설정 없이, 프로젝트 안에 계획 파일 하나만 만들어도 같은 원리를 적용할 수 있겠다고 생각했다.

그래서 만든 게 **PLAN.md** 파일이다.

```
# 인천이벤트 사이트 성장 계획

## 1단계 — 측정 기반 만들기
### [ ] Google Analytics 연결
### [ ] Google Search Console 등록
...

## 진행 현황
| 항목 | 상태 |
| Google Analytics 연결 | ✅ 완료 |
```

이 파일을 프로젝트에 만들어두고, Claude Code가 항상 이 파일을 참조하도록 했다. 세션이 초기화돼도 계획 파일을 보면 어디까지 했는지 바로 파악할 수 있다. 일종의 **AI를 위한 인수인계서**다.

## 어떻게 진행했나

### 1. 주제 정하기

인천에 살면서 행사 정보를 찾기가 너무 불편했다. 시청 홈페이지, 구청 홈페이지, 각종 기관 사이트를 따로따로 들어가야 하는 게 귀찮았다. 이걸 한 곳에 모아주는 사이트를 만들어보자고 생각했다.

### 2. 요구사항 정리하기

- 인천의 재미있는 행사를 한눈에 볼 수 있는 사이트
- 서버 비용 내기 싫으니 무료 배포
- 단순하게 1페이지짜리 화면
- 데이터는 크롤링으로, 나중엔 제보도 받고 싶다

### 3. PLAN.md 만들기

Claude Code한테 아래처럼 질문했다.

```
인천의 행사를 한번에 보여주는 사이트를 만들고싶어.
요구사항
- 인천의 재미있는 행사를 한번에 보는 사이트를 만들어줘
- 나 돈이 없어, AWS에 못 올리니까 깃허브 페이지로 무료로 올리자
- 이왕이면 많은 사람들이 볼 수 있게 하고싶어
- 데이터는 내가 손으로 다 작성할 수 없으니, 인천 시청이나 이벤트 홈페이지에서 크롤링하자
- 어느정도 데이터가 쌓이면 제보가 들어올 수 있으니 그건 깃허브 이슈로 받자

계획을 세워서 plan.md로 저장해줘
```

그랬더니 Claude Code가 단계별 계획을 알아서 세우고 PLAN.md 파일까지 만들어줬다.

### 4. PLAN.md 순서대로 진행하기

여기서부터는 사실 **승인/거절**밖에 한 게 없다. Claude가 뭔가를 제안하면 보고, 괜찮으면 승인하고, 이상하면 거절하고 방향을 다시 잡아줬다. 중간중간 결과물을 확인하면서 이상한 부분을 짚어주고, 필요하면 PLAN.md도 같이 수정해가면서 진행했다.

구체적으로 어떻게 진행됐는지 단계별로 정리하면 이렇다.


## 느낀 점

**잘 된 것들**
- 정말로 코딩 없이 완성됐다
- 막히면 이유를 설명해준다 ("이게 왜 404가 나냐"고 물어보면 진짜로 원인을 찾아줬다)
- PLAN.md 덕분에 며칠에 걸쳐 끊어서 작업해도 문맥이 유지됐다

**아직 한계도 있다**
- AI가 항상 맞는 건 아니다. GA 인증 실패한 것처럼 첫 시도가 틀릴 때도 있다
- "왜 실패했는지" 원인 분석은 여전히 사람이 확인해야 한다
- AI는 스스로 생각하는 게 아니라, 고도화된 통계 머신이다. 앞 맥락이 명확해야 뒷 대답이 정확하다. 결국 정확히 요구해야 정확히 대답해준다
